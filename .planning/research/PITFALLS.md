# Pitfalls Research: v1.2 Notes, Browser & Platform Growth

**Researched:** 2026-04-09

## Pitfall 1: Letting Notes scope explode into sync and editor work

**Risk:** Search and organization turn into backend sync, collaboration, or rich-text scope.

**Prevention:**
- keep `v1.2` Notes local-first
- defer sync/accounts and rich-text editing explicitly

## Pitfall 2: Overclaiming Browser capability

**Risk:** Direct URL entry gets framed as “full browser support” when many sites will still require external fallback.

**Prevention:**
- keep explicit embed vs external behavior in the model
- keep product copy honest about limits

## Pitfall 3: Turning platform groundwork into a full marketplace too early

**Risk:** Submission/catalog work expands into trust systems, moderation, pricing, or install escape hatches.

**Prevention:**
- scope `v1.2` to repo-driven submission and browsing surfaces only
- defer arbitrary install flows

## Pitfall 4: Losing the launcher-path quality bar

**Risk:** Feature work is verified only at helper/module level while real app flows regress.

**Prevention:**
- end the milestone with launcher-path verification for Notes, Browser, and the new catalog surfaces
